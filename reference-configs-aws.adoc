AWS 支援多種 Cloud Volumes ONTAP 配置。



== 支援的節點數量

Cloud Volumes ONTAP 在 AWS 中既可以作為單節點系統使用，也可以作為高可用性（HA）節點對使用，以實現容錯和不中斷營運。

不支援將單節點系統升級為 HA 配對。如果您想要在單節點系統和 HA 配對之間切換，則需要部署新系統，並將資料從現有系統複寫到新系統。



== 支援的儲存設備

Cloud Volumes ONTAP 支援多種類型的 EBS 磁碟和 S3 物件儲存，用於資料分層。最大儲存容量取決於您選擇的授權。



=== 依授權提供的儲存支援

每個授權支援不同的最大系統容量。最大系統容量包括磁碟型儲存設備以及用於資料分層的物件儲存設備。NetApp 不支援超過此限制。

. AWS 本機區域中的 Cloud Volumes ONTAP 組態僅支援通用型 SSD（gp2）磁碟類型。AWS 本機區域中的 Cloud Volumes ONTAP 不支援其他磁碟類型。

== 支援的 EC2 運算

每個 Cloud Volumes ONTAP 授權支援不同的 EC2 執行個體類型。為方便起見，下表列出了每種受支援的執行個體類型的 vCPU、RAM 和頻寬 https://aws.amazon.com/ec2/instance-types/["有關 EC2 執行個體類型的最新完整詳細資訊，請參閱 AWS。"^]。

Cloud Volumes ONTAP 可以在預留或隨選 EC2 執行個體上執行。不支援使用其他執行個體類型的解決方案。

下表所示的頻寬與每種執行個體類型的 AWS 文件化限制相符。這些限制與 Cloud Volumes ONTAP 所能提供的效能並不完全一致。有關預期效能，請參閱 https://www.netapp.com/pdf.html?item=/media/9088-tr4383pdf.pdf["NetApp 技術報告 4383：Amazon Web Services 中 Cloud Volumes ONTAP 在應用工作負載下的效能特性分析"^]。

. 某些執行個體類型包含本機 NVMe 儲存設備， Cloud Volumes ONTAP 將其用作 _Flash Cache_ 。Flash Cache 透過對最近讀取的使用者資料和 NetApp 中繼資料進行即時智慧快取，加快資料存取速度。它對隨機讀取密集型工作負載（包括資料庫、電子郵件和檔案服務）非常有效。若要利用 Flash Cache 效能提升，必須在所有磁碟區上停用壓縮功能 https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-flash-cache.html["深入瞭解 Flash Cache"^]。
. 使用 HA 配對時，Cloud Volumes ONTAP 支援大多數執行個體類型的高寫入速度。使用單一節點系統時，所有執行個體類型均支援高寫入速度 https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-write-speed.html["深入瞭解如何選擇寫入速度"^]。
. r5.12xlarge 執行個體類型存在已知的支援性限制。如果節點因當機而意外重新開機、系統可能無法收集用於疑難排解和根本原因分析的核心檔案。客戶接受風險和有限的支援條款、並在發生此情況時承擔所有支援責任。此限制會影響新部署的 HA 配對、以及從 9.8 升級的 HA 配對。此限制不會影響新部署的單一節點系統。
. 雖然這些 EC2 執行個體類型支援超過 64 個 vCPU，Cloud Volumes ONTAP 最多只支援 64 個 vCPU。
. 選擇 EC2 執行個體類型時、您可以指定是共用執行個體還是專用執行個體。
. AWS Local Zones 支援以下 EC2 執行個體類型系列，大小從 xlarge 到 4xlarge：M5、C5、C5d、R5 和 R5d。link:https://aws.amazon.com/about-aws/global-infrastructure/localzones/features/?nc=sn&loc=2["有關本地區域中支援的 EC2 執行個體類型的最新完整詳細資訊，請參閱 AWS"^]。
+
AWS Local Zones 中的這些執行個體類型不支援高寫入速度。





=== c4 、 m4 和 r4 執行個體不再受支援

Cloud Volumes ONTAP 已不再支援 AWS 中的 c4、m4 和 r4 EC2 執行個體類型。如果您的系統在 c4、m4 或 r4 執行個體上執行，請變更為 c5、m5 或 r5 執行個體。在變更執行個體類型之前，您無法升級至此版本。

link:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-change-ec2-instance.html["了解如何變更 Cloud Volumes ONTAP 的 EC2 執行個體類型"^]。

如需更多資訊，請參閱：

* https://kb.netapp.com/Cloud/Cloud_Volumes_ONTAP/Converting_an_AWS_Xen_CVO_instance_to_Nitro_KVM["知識庫（KB）文章：將 AWS Xen CVO 執行個體轉換為 Nitro KVM"^]
* https://kb.netapp.com/Cloud/Cloud_Volumes_ONTAP/Unable_to_change_the_instance_type_from_r4_to_r5_with_disk_count_error["KB 文章：磁碟計數錯誤，無法將執行個體類型從 r4 變更為 r5"^]
* link:https://mysupport.netapp.com/info/communications/ECMLP2880231.html["深入瞭解這些執行個體類型的可用性和支援終止"^]
